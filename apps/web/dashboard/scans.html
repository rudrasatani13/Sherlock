Future scan flow

How scanning will work.

  1. 1. Verified target

    Target ownership is confirmed via DNS, meta tag, well-known file, or manual review.

  2. 2. Scan job queued

    Authenticated user creates a scan job. Safety gates validate the request before queuing.

  3. 3. Worker runs scanner

    Background worker picks up the job and executes the Phase 5 scanner engine.

  4. 4. Evaluator reviews

    Phase 7 evaluator classifies scanner observations into structured findings.

  5. 5. Findings and report

    Reviewed findings are presented in a web report with evidence and remediation.

Job lifecycle states

Queue status reference.

Status Description
Queued Job is waiting in the queue for a worker
Running Worker has picked up the job and is executing
Completed Job finished successfully
Failed Job encountered an error during execution
Cancelled Job was cancelled before completion
Timed Out Job exceeded its timeout limit
Blocked — Unverified Target ownership is not verified
Blocked — Unsafe Safety gate rejected the job
