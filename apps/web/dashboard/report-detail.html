Executive summary

Security posture for the tested demo scope.

Sherlock tested selected AI launch security categories for a fictional RAG support copilot using sanitized demo findings. The report focuses on observed behavior, business impact, evidence quality, and practical fixes.

The most important demo risks are restricted retrieval content entering answers, tool actions being prepared without a hard permission boundary, and generated content rendering in a way that could mislead users.

This report does not claim the target is fully protected. It only communicates what was tested, what was observed, what was not tested, and what needs review.

Top fixes

Recommended remediation order.

  1. Enforce document authorization before retrieval.

    Filter by tenant, role, document permission, and source trust before content enters model context. Retest with synthetic canaries and negative controls.

  2. Add server-side tool permission checks.

    Separate draft and execute actions, validate tool arguments, and require explicit confirmation for risky workflows.

  3. Sanitize generated output before rendering.

    Escape HTML, allowlist Markdown, block active links, and visually separate AI-generated suggestions from trusted product actions.
