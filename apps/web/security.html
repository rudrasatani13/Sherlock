Security and trust

Sherlock should be careful before it scans anything real.

PowerDetect Sherlock is meant to test sensitive AI systems. Public scanning must ship only after ownership verification, abuse prevention, data handling, and reporting controls are designed and implemented.

Ownership verification

Sherlock will require proof that the requester owns or is authorized to test the target before public scanning is allowed.

Secret handling

API keys, provider tokens, credentials, and private keys should not be stored in plain text. Managed secret storage should be used in deployed environments.

Protected scan logs

Scan logs may contain sensitive data, prompts, responses, headers, files, or customer context. Logs need minimization, access control, retention policy, and redaction.

Redacted reports

Reports should preserve remediation value while redacting secrets, personal data, customer identifiers, tokens, and sensitive evidence.

No complete-security claim

Passing a future Sherlock scan does not guarantee complete security. Sherlock should describe observed behavior, tested categories, assumptions, and limitations.

Abuse prevention before scanning

Rate limits, cancellation, timeouts, spend controls, target verification, and SSRF protection are required before public scan creation exists.

Network safety

Public scanners can become attack infrastructure if controls are missing.

  • Block private, internal, link-local, loopback, and metadata-service IP ranges before URL scanning.
  • Resolve and validate redirects, DNS rebinding behavior, and user-supplied hostnames.
  • Enforce timeouts, body limits, retry limits, concurrency limits, and spend limits.
  • Keep scan target verification auditable before any external test is allowed.

Reporting language

Reports should help teams fix risk without overclaiming.

Say what was observed

Findings should be grounded in behavior, evidence, affected surface, and impact.

Say what was not tested

Limitations, assumptions, missing access, and out-of-scope systems should be explicit.

Avoid certification language

Sherlock should not claim that a target is fully secure or protected against every attack.
